# Тест-кейсы для проверки микросервиса

## 1 Проверка диапазона значений sellerId [Task2](test_1.py)
1.1 Проверка диапазона допустимых значений
1) Отправить POST-запрос на создание объявления с sellerId 111111 
2) Отправить POST-запрос на создание объявления с sellerId 999999
3) Отправить POST-запрос на создание объявления с sellerId 499999
Ожидаемый результат:
- Выдает статус 200
- В ответе присутствует уникальный id объявления
1.2 Проверка диапазона недопустимых значений
1) Отправить POST-запрос на создание объявления с sellerId 111110
2) Отправить POST-запрос на создание объявления с sellerId 1000000
Ожидаемый результат:
- Выдает статус 400
- Возвращается ошибка валидации

## 2 Создание объявления (POST) [Task2](test_2.py)
Предусловие:
1) Сгенерировать уникальный sellerId (17092025)
2) Отправить POST запрос на /ads с данными объявления  
2.1 Успешное создание с корректными данными.
Создать объявление с этими данными.
Ожидаемый результат:
- Выдает статус 200
- В ответе присутствует уникальный id объявления

2.2 Попытка создания с некорректными данными
Создание объявления из предусловия с некорректными данными (например, пустые поля, невалидный sellerID).
Ожидаемый результат:
- Статус ответа 400 Bad Request
- Сообщение об ошибке о некорректном sellerId.
