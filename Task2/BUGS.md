##
**1 Проверка диапазона значений sellerId**
bug_1. [Валидация диапазона sellerID отсутствует при создании объявления]

Приоритет: Low

Предусловие:
API доступно по адресу
https://qa-internship.avito.com/api/1/item

Шаги воспроизведения:
Отправить POST-запрос на создание объявления с телом:
{
  "sellerID": 111110,
  "name": "Тестовое объявление",
  "price": 10000,
  "statistics": {
    "likes": 10,
    "viewCount": 20,
    "contacts": 5
  }
}
Отправить POST-запрос на создание объявления с телом:
{
  "sellerID": 1000000,
  "name": "Тестовое объявление",
  "price": 10000,
  "statistics": {
    "likes": 10,
    "viewCount": 20,
    "contacts": 5
  }
}

Ожидаемый результат:
- API возвращает статус 400 Bad Request
- Возвращается ошибка валидации диапазона sellerID
- Объявление не создаётся

Фактический результат:
- API возвращает статус 200 OK
- Объявление успешно создаётся
- В ответе возвращается сообщение с id
- Валидация диапазона sellerID отсутствует  


**2. Создание объявления** 
bug_2. [При отправке POST-запроса на создание объявления с корректными данными сервер возвращает HTTP статус 200 OK]

Приоритет: medium

Предусловие:
API доступно по адресу
https://qa-internship.avito.com/api/1/item

Шаги воспроизведения:
- Отправить POST-запрос на `https://qa-internship.avito.com/api/1/item` с валидными данными объявления (например, уникальный `sellerID`, `name`, `price`, `statistics`).
- Посмотреть ответ сервера.
Ожидаемый результат: 
Статус ответа — 201 Created.

Фактический результат: 
Статус ответа — 200 OK.

Примечания:иВ теле ответа присутствует ID созданного объявления и прочие данные, всё работает функционально, но статус код не соответствует стандарту.
